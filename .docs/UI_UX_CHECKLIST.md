# UI/UX Checklist

이 문서는 AR 사용자 경험 관련 문제점과 디자인 변경 방향을 체크박스 형식으로 누적 관리하기 위한 문서다.

## Current Scenario

- 사용자가 알고 싶은 건물을 휴대폰 화면 정중앙으로 가져다대야 상단에 간단 라벨이 생긴다.
- 사용자가 상단 라벨을 클릭하면 하단에서 상세정보 패널이 올라오며 정보를 확인할 수 있다.

## Problems

- [ ] 거리가 있는 위치에서 건물이 촘촘한 경우, 미세한 흔들림만으로도 중앙 포커스를 잃어 라벨 대상이 자주 바뀐다.
- [ ] 상세정보 진입 버튼이 상단에 있어 한 손으로 기기를 고정한 상태에서 누르기 어렵다.
- [x] 상세정보 패널을 열어도 2D 마커가 더 위에 보여 상세정보보다 먼저 인지된다.

## Design Changes

- [x] 2D 마커를 현재 형태에서 보기 편안한 핀 모양으로 변경한다.
- [x] 상세정보 창을 PNG 이미지 의존형 구조에서 더 자연스러운 UI 컴포넌트 기반 구조로 개선한다.
- [x] 상세정보 패널을 UI Toolkit 기반 구조로 전환한다.
- [x] 상세정보 패널은 `건물명`, `주소`, `전화번호`, `카카오맵 연동` 중심의 정보 구조만 유지한다.
- [x] 상세정보 패널을 AR 화면 위에 붙는 바텀시트 형태로 재구성한다.
- [x] 상세정보 패널은 화면 하단 80%를 차지하고, 상단 20%는 AR 화면이 남도록 조정한다.
- [x] 상세정보 패널의 좌우/하단은 화면을 꽉 채우고, 스크롤바는 숨겨 더 앱다운 인상을 만든다.

## Progress Log

### 1. 초기 문제 정의

- [x] 현재 시나리오를 `건물 중앙 포커스 -> 상단 라벨 생성 -> 라벨 선택 -> 하단 상세정보 노출` 흐름으로 정리했다.
- [x] 포커스 흔들림, 상단 터치 불편, 상세정보보다 위에 보이는 2D 마커 문제를 핵심 이슈로 정의했다.

### 2. 마커 방향 수정

- [x] 기존 사람 모양 느낌의 마커를 핀 형태로 바꾸는 방향을 확정했다.
- [x] 2D 마커를 핀 형태로 변경하고, 더 눈에 띄는 빨간색 계열로 조정했다.
- [x] 그림자와 레이어를 활용해 3D 오브젝트는 아니지만 입체감이 느껴지도록 마커를 보정했다.

### 3. 상세정보 패널 구조 재검토

- [x] 기존 상세정보 패널이 `diagram` 폴더의 PNG 이미지 의존형 구조라는 점을 확인했다.
- [x] PNG를 제거하고 코드형 카드 UI로 1차 전환을 진행했다.
- [x] 코드형 카드만으로는 기존 씬 구조 한계 때문에 UI가 딱딱하고 어색하다는 점을 확인했다.
- [x] 상세정보 패널은 AR 마커와 달리 `UI Toolkit`으로 분리할 수 있다는 방향을 확정했다.

### 4. UI Toolkit 도입

- [x] UI Toolkit용 `UIDocument`, `UXML`, `USS`, 컨트롤러 스크립트 기반 구조를 프로젝트에 추가했다.
- [x] `Tools > AR > Setup Detail Panel UI Toolkit` 메뉴로 세팅할 수 있게 구성했다.
- [x] `ARUIManager`와 UI Toolkit 상세 패널을 연결해 기존 uGUI 상세 패널 대신 새 패널이 열리도록 구성했다.
- [x] 기존 PNG 기반 상세 패널이 UI Toolkit 패널 위에 겹쳐 보이는 문제를 수정해, UI Toolkit 패널만 보이도록 정리했다.

### 5. 정보 구조 단순화

- [x] 상세정보 항목을 `건물명`, `주소`, `전화번호`, `카카오맵 연동` 4개로 압축했다.
- [x] 주요시설 등 부가 정보는 현재 상세 패널 범위에서 제거하고, 핵심 정보 우선 구조로 정리했다.
- [x] 잘못된 폰트 경고와 불필요한 스타일 요소를 줄여 UI Toolkit 패널의 기본 동작을 안정화했다.

### 6. 바텀시트형 UI 개선

- [x] 새 페이지 전환형보다 AR 문맥을 유지하는 바텀시트형이 더 적합하다고 판단했다.
- [x] 상세정보 패널이 아래에서 위로 등장하는 애니메이션을 적용했다.
- [x] 휴대폰 화면 기준 약 80% 높이의 바텀시트로 조정했다.
- [x] 제목, 주소, 전화번호, 버튼 크기와 줄간격을 조정해 텍스트 겹침과 과도한 두께감을 완화했다.
- [x] 좌우와 하단이 비어 보이지 않도록 풀폭 바텀시트 형태로 수정했다.
- [x] 상단 약 20%는 AR 화면이 계속 보이도록 유지했다.
- [x] UI Toolkit 스크롤바를 숨겨 시각적으로 더 자연스럽게 정리했다.

### 7. 현재 상태

- [x] 마커는 핀 형태로 변경된 상태다.
- [x] 상세정보는 PNG 기반이 아니라 UI Toolkit 바텀시트 기반으로 전환된 상태다.
- [ ] 실제 기기 기준으로 긴 건물명 줄바꿈, 세이프에어리어 하단 버튼 여백, 최종 시각 밀도는 추가 미세 조정이 필요하다.

## Notes

- 새 문제점이 생기면 `Problems` 섹션에 체크박스로 추가한다.
- 새 디자인 변경안이 생기면 `Design Changes` 섹션에 체크박스로 추가한다.
- 완료된 항목은 `[x]`로 변경한다.
