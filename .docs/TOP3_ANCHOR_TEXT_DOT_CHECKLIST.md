# 화면 내 Top3 앵커 / 중앙 텍스트 / 양옆 점 체크리스트

## 목표

- 화면 안에 들어온 건물 중 최대 3개까지 앵커를 동시에 유지한다.
- 화면 중앙에 가장 가까운 건물 1개는 건물 정보 텍스트로 표시한다.
- 중앙에 있지 않지만 화면 안에 있는 나머지 건물은 점 형태 마커로 표시한다.
- 사용자가 점을 화면 중앙으로 가져오면 해당 건물 정보가 텍스트로 전환되는 흐름을 만든다.

## 1단계. 후보 계산과 표시 로직 분리

- [x] `GeospatialManager`에서 "화면 안 후보 계산"과 "실제 표시 상태 결정"을 분리한다.
- [x] 현재 `focus 1개` 중심 흐름을 확인하고, `visible 후보 목록`을 별도로 유지한다.
- [x] 디버그 로그에 `visible count`, `visible list`, `focused candidate`를 따로 남긴다.

## 2단계. 화면 안 Top3 선정 로직 추가

- [x] `GetVisibleBuildingCandidates()` 결과에서 실제 표시 대상으로 쓸 상위 3개를 고르는 단계를 만든다.
- [x] `GetFrontMostCandidates()`가 후보를 과하게 제거하지 않는지 점검한다.
- [x] `FindBestTargetFromExistingAnchors()`는 "가운데 1개 focus"용으로만 사용하도록 역할을 줄인다.
- [x] 최종적으로 `top3 candidates`와 `focused candidate`를 별도로 구분한다.

## 3단계. 단일 앵커 구조를 3-anchor pool 구조로 변경

- [x] 기존의 "선택된 1개만 anchor 생성" 흐름을 찾는다.
- [x] top3에 포함된 건물은 동시에 앵커를 유지하도록 변경한다.
- [x] top3에서 빠진 건물만 제거하고, 새로 들어온 건물만 생성하도록 수정한다.
- [x] 매 프레임 전체 anchor를 지우고 다시 만드는 구조가 남아 있지 않은지 확인한다.

## 4단계. 마커 상태를 Text / Dot로 분리

- [x] `BuildingMarker` 상태를 `Focused(Text)` / `Preview(Dot)` / `Hidden` 중심으로 재정의한다.
- [x] 가운데 건물은 텍스트, 양옆 건물은 점으로 보이도록 상태 전환 규칙을 만든다.
- [x] 상세정보 진입은 `Focused(Text)` 상태일 때만 가능하도록 유지한다.

## 5단계. BuildingMarker 시각 구조 분리

- [x] `BuildingMarker` 안에 텍스트 visual과 점 visual을 각각 만든다.
- [x] 텍스트 상태일 때는 점을 숨기고, 점 상태일 때는 텍스트를 숨기도록 한다.
- [x] 점은 멀리서도 구분되도록 충분히 단순하고 명확한 형태로 유지한다.
- [ ] 텍스트 버튼 배경, 글자 크기, collider가 `Focused` 상태에서만 활성화되도록 정리한다.

## 6단계. Focus 전환 안정화

- [ ] 화면 중앙에 더 가까운 건물이 생겨도 즉시 전환하지 않도록 히스테리시스 규칙을 추가한다.
- [ ] 새 focus가 일정 시간 유지되거나 점수 차가 충분할 때만 텍스트 전환하도록 한다.
- [ ] 애매하게 1자 형태로 겹친 건물 사이에서 focus가 빠르게 흔들리지 않는지 확인한다.

## 7단계. 배치 규칙 분리

- [ ] `Focused(Text)`와 `Preview(Dot)`가 같은 배치 규칙을 쓰지 않도록 분리한다.
- [ ] 텍스트는 읽기 쉬운 위치, 점은 건물 방향을 암시하는 위치에 배치되도록 조정한다.
- [ ] 점이 너무 사용자 쪽으로 나오지 않게 하고, 텍스트는 지나치게 UI처럼 보이지 않도록 균형을 맞춘다.

## 8단계. 상세정보 버튼 연동 정리

- [ ] 하단 상세정보 버튼은 현재 `Focused(Text)` 건물에만 연결되도록 유지한다.
- [ ] `Preview(Dot)` 상태 건물은 점만 보이고 상세정보 대상은 되지 않게 한다.
- [ ] focus 전환 시 하단 상세정보 대상도 정확히 같이 바뀌는지 확인한다.

## 9단계. 디버그 및 검증 로그 추가

- [ ] 디버그 오버레이에 `visible count`, `top3`, `focused`, `active anchors`, `marker state`를 출력한다.
- [ ] "현재 3개가 잡혔는지", "가운데 1개만 텍스트인지", "양옆이 점인지"를 로그만 보고도 판단 가능하게 만든다.
- [ ] 전환 시점에 어떤 후보가 빠지고 들어왔는지 로그로 추적 가능하게 한다.

## 10단계. 권장 구현 순서

- [ ] 1차: top3 후보 선정
- [ ] 2차: 3-anchor pool 유지
- [ ] 3차: 가운데 1개 focus 확정
- [ ] 4차: 양옆 2개 dot 표시
- [ ] 5차: focus hysteresis 추가
- [ ] 6차: 텍스트/점 배치 튜닝

## 체크 포인트

- [ ] 화면 안 건물 3개가 동시에 들어왔을 때 앵커가 최대 3개 유지되는가
- [ ] 화면 중앙 건물만 텍스트로 보이는가
- [ ] 중앙이 아닌 건물은 점으로만 보이는가
- [ ] 사용자가 좌우로 카메라를 틀면 점이 텍스트로 자연스럽게 전환되는가
- [ ] 텍스트/점 전환이 너무 쉽게 흔들리지 않는가
- [ ] 대각선 구도에서도 사용자가 "저 점을 중앙에 맞추면 텍스트가 뜨겠구나"라고 이해할 수 있는가
