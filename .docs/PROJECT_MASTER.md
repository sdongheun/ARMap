## 2. 프로젝트 한 줄 정의

> 이 프로젝트는 현장 사용자가 스마트폰 카메라로 주변 건물 또는 시설을 직관적으로 탐색할 수 있도록, 위치 기반 Geospatial AR 마커와 상세 정보를 제공하는 모바일 AR 안내 서비스를 목표로 한다.

## 3. 배경과 문제 정의

### 3.1 현재 상황

- 프로젝트는 Unity AR Foundation, ARCore Extensions, ARKit 설정이 포함된 모바일 AR 앱 형태로 구성되어 있다.
- 지리공간 앵커를 활용해 실제 공간 좌표 위에 AR 마커를 띄우는 구조가 구현되어 있다.
- 카카오 로컬 API를 이용해 주변 장소 정보를 수집하고, 이를 건물 또는 시설 단위로 묶어 사용자에게 보여주는 흐름이 구현되어 있다.
- iOS 빌드 산출물이 함께 존재하는 것으로 보아, 최소한 iOS 배포 또는 테스트를 목표로 진행 중인 상태로 추정된다.

### 3.2 사용자가 겪는 문제

- 실제 공간에서 주변 건물이나 입점 시설 정보를 빠르게 파악하기 어렵다.
- 일반 지도 앱은 방향성과 현장 시야 기반 탐색 경험이 약하다.
- 특정 건물 또는 시설 정보를 찾기 위해 지도 검색, 상세 진입, 전화/주소 복사 등 여러 단계를 거쳐야 한다.

### 3.3 왜 이 프로젝트가 필요한가

- 사용자가 카메라를 비춘 방향 기준으로 주변 건물과 시설을 바로 인지할 수 있어야 한다.
- 위치와 시야를 결합한 AR 탐색 방식은 지도 중심 UX보다 현장성 있는 안내에 적합하다.
- 향후 캠퍼스, 상권, 전시 공간, 관광 안내 등으로 확장 가능한 기반 구조를 확보할 수 있다.

## 4. 목표

### 4.1 최종 목표

- 1. 사용자가 현재 위치에서 주변 건물 또는 시설을 AR 마커로 확인하고, 선택한 대상의 핵심 정보와 상세 정보를 즉시 확인할 수 있는 모바일 앱을 완성한다.
- 2. 위 1번 목표를 달성한 뒤 ar 길찾기 시스템을 추가해 사용자가 현재 위치에서 목표 위치까지 목적지를 설정하면 화면 상에 ar 안내 화살표를 구현해 사용자가 가는 길을 안내한다.

### 4.2 세부 목표

- GPS 및 Geospatial Anchor를 기반으로 주변 주요 장소를 실제 공간에 정렬해 표시한다.
- 내가 보는 방향 건물 뒤에 있는 건물은 인식해선 안된다. (내가 보고 있는 건물의 대한 정보만 알려줘야 한다.)
- 카메라 시야 중심의 건물을 자동 감지하고 빠르게 요약 정보를 노출한다.
- 카메라 시야에는 있지만 중앙에 있지 않은 건물의 정보는 미제공하고, 마커 형태로 존재해서 사용자가 화면 중앙에 그 마커로 가져다 놓으면 해당 마커의 요약된 정보가 나타나야한다.
- 상세 패널에서 주소, 전화, 지도 링크, 입점 시설 목록까지 확인할 수 있게 한다.
- 일정 거리 이상 이동 시 주변 데이터를 다시 불러와 현 위치 기준 정보로 갱신한다.

## 7. 핵심 사용자 시나리오

### 시나리오 1. 주변 건물 탐색

- 상황: 사용자가 현장에서 앱을 실행한다.
- 사용자의 목표: 주변 건물 중 카메라가 향하는 방향의 건물을 빠르게 파악한다.
- 진행 흐름:
  1. 앱이 위치 권한을 확인하고 현재 위치를 가져온다.
  2. 앱이 주변 장소 데이터를 조회하고 건물 단위로 정리한다.
  3. 앱이 Geospatial Anchor를 생성하고 AR 마커를 배치한다.
  4. 사용자가 카메라를 움직이면 시야에 있는 건물을 자동 감지한다.
  5. 감지된 건물의 중 화면 정중앙은 요약 정보가 카드 형태로 표시, 정중앙이 아닌 건물은 마커로 표시된다.
- 기대 결과: 사용자는 지도 검색 없이 현재 시야 기준의 장소 정보를 확인할 수 있다.

### 시나리오 2. 상세 정보 확인

- 상황: 사용자가 감지된 건물 정보를 더 자세히 보고 싶다.
- 사용자의 목표: 주소, 전화번호, 지도 링크, 입점 시설 정보를 확인한다.
- 진행 흐름:
  1. 사용자가 감지된 건물의 요약 정보 상태에서 액션 버튼(미정)을 누른다.
  2. 앱이 퀵 정보 카드를 표시한다.
  3. 사용자가 상세 보기 버튼을 누른다.
  4. 앱이 상세 패널을 열고 건물 및 시설 정보를 보여준다.
  5. 사용자가 주소 복사, 전화 연결, 지도 열기 중 원하는 행동을 수행한다.
- 기대 결과: 사용자는 외부 검색 없이 필요한 주요 정보를 즉시 활용할 수 있다.

### 시나리오 3. 위치 이동 후 재탐색

- 상황: 사용자가 다른 위치로 이동한다.
- 사용자의 목표: 새 위치 기준의 주변 장소 정보를 자동으로 다시 받는다.
- 진행 흐름:
  1. 앱이 현재 위치와 마지막 조회 위치를 비교한다.
  2. 설정 거리 이상 이동하면 장소 데이터를 다시 요청한다.
  3. 기존 앵커를 정리하고 새 앵커 및 마커를 생성한다.
- 기대 결과: 사용자는 위치가 바뀌어도 최신 주변 정보를 계속 탐색할 수 있다.

## 9. 비기능 요구사항

### 9.1 성능

- 초기 위치 확보 후 첫 장소 로드는 가능한 한 5초 이내에 시작되어야 한다.
- 주변 장소 조회 및 앵커 재생성은 사용자 체감상 지연이 과도하지 않아야 한다.
- 감지 로직과 마커 갱신은 프레임 드랍을 유발하지 않는 수준으로 유지되어야 한다.

### 9.2 안정성

- 위치 서비스 실패 시 앱이 비정상 종료되지 않아야 한다.
- 외부 API 실패 시 최소한 재시도 또는 실패 상태 안내가 필요하다.
- 앵커 생성 실패 시 전체 기능이 멈추지 않고 일부 대상만 제외 처리되어야 한다.
- 밀집된 거리와 밀집되지 않은 거리에서의 건물 정보 데이터를 불러오는 개수를 최적화해야 한다.

### 9.3 보안

- 카카오 REST API 키와 ARCore Cloud Services 키는 소스 코드 하드코딩보다 로컬 파일 또는 빌드 설정 기반 주입을 우선한다.
- 민감한 API 키 파일은 저장소 커밋 대상에서 제외되어야 한다.
- 외부 URL 호출은 신뢰 가능한 대상만 사용해야 한다.

### 9.4 운영 및 유지보수

- 주요 이벤트 로그 기준이 필요하다.
- 최소 로그 대상은 위치 서비스 시작/실패, API 호출 성공/실패, 앵커 생성 수, 상세 진입 수다.
- iOS 빌드 후처리와 API 키 동기화 스크립트는 재현 가능해야 한다.

| 월드 마커      | 실제 공간 정렬 안내      | 3D 마커, 월드 라벨                 | 시선 정렬                   | 선택 대상 강조                    |
| 상세 패널      | 세부 정보 확인           | 주소, 전화, 시설 리스트, 액션 버튼 | 복사, 전화, 지도 열기       | 하단 시트 형태로 추정             |

## 11. 기술 구조

### 11.1 사용 기술

- 클라이언트: Unity
- AR 프레임워크: AR Foundation, ARCore Extensions, ARKit
- 지리공간 기능: ARCore Geospatial API
- 외부 데이터 소스: 카카오 로컬 API
- UI: Unity UI, TextMesh Pro
- iOS 의존성: CocoaPods 기반 ARCore/Firebase 포함

### 11.2 아키텍처 메모

- `GeospatialManager`가 위치 획득, 외부 API 호출, 데이터 가공, 앵커 생성, 감지 로직을 총괄한다.
- `ARUIManager`가 상태 카드, 퀵 정보, 상세 패널, 토스트, 화면 마커 표시를 담당한다.
- `BuildingData`와 `FacilityInfo`가 장소/시설 데이터 모델 역할을 한다.
- `BuildingMarker`가 월드 공간 마커의 시각 상태와 정보 라벨을 담당한다.
- 현재 구조는 프로토타입에는 적합하지만, 장기적으로는 데이터 수집, 감지, UI, 플랫폼 의존 처리를 분리하는 리팩터링이 필요하다.

- 가능하면 Android 실제 기기 테스트

### 16.2 검증 체크리스트

- [ ] 위치 권한 허용 시 초기 데이터가 정상 로드된다.
- [ ] 위치 권한 거부 시 오류 없이 안내된다.
- [ ] 주변 장소가 중복 없이 표시된다.
- [ ] 감지 대상이 시야 중심 기준으로 자연스럽게 바뀐다.
- [ ] 선택 건물과 주변 건물의 마커 상태가 구분된다.
- [ ] 퀵 정보 카드와 상세 패널 내용이 일관된다.
- [ ] 주소 복사, 전화 연결, 지도 열기가 정상 동작한다.
- [ ] 일정 거리 이상 이동 시 데이터가 갱신된다.
- [ ] 외부 API 실패 시 앱이 멈추지 않는다.
