# 구현 체크리스트

## 목표

- [ ] 건물 탐지/표시 문제를 한 번에 전부 수정하지 않는다.
- [ ] 작은 단위로 구현하고, 각 단계마다 실제 기기 테스트를 거친 뒤 다음 단계로 넘어간다.
- [ ] 매 단계는 원인 하나 또는 표시 정책 하나만 다루도록 제한한다.

## 작업 원칙

- [ ] 한 단계에서 해결하려는 문제를 하나로 좁힌다.
- [ ] 코드 수정 후 바로 다음 단계로 넘어가지 않고 먼저 사용자 검수를 받는다.
- [ ] 검수 결과에 따라 다음 단계의 범위를 다시 조정할 수 있게 유지한다.
- [ ] 큰 구조 변경은 앞 단계 검수가 끝난 뒤에만 진행한다.

## 0단계. 기준 증상 고정

- [ ] `15m 내외에서 건물을 보고 있어도 점/텍스트가 안 보이는 증상`을 기준 현상으로 고정한다.
- [ ] `가로모드에서 두 건물을 한 화면에 담아도 중앙 텍스트 + 주변 점이 안정적으로 안 보이는 증상`을 기준 현상으로 고정한다.
- [ ] 이후 모든 수정은 위 두 증상 중 어느 쪽을 줄이기 위한 것인지 명시한다.

## 1단계. 점이 아예 안 보이는 문제부터 수정

### 구현 범위

- [ ] `topCandidates`에 들지 못해 점조차 안 보이는 문제를 먼저 줄인다.
- [ ] 화면 안에 있는 후보 건물이 최소한 점으로 보일 수 있도록 표시 조건을 재정리한다.
- [ ] 이 단계에서는 텍스트 정책을 크게 바꾸지 않는다.
- [ ] 이 단계에서는 가로모드 전용 로직을 본격적으로 건드리지 않는다.

### 구현 후 사용자 검수

- [ ] 사용자가 세로모드에서 15m 전후 거리로 테스트한다.
- [ ] 건물이 화면 안에 들어와 있을 때 최소한 점이 보이는지 확인받는다.
- [ ] 점이 여전히 안 보이는 조건을 다시 기록한다.
- [ ] 사용자 검수 전까지 2단계로 넘어가지 않는다.

## 2단계. 중앙 1개 텍스트 + 주변 점 구조 정리

### 구현 범위

- [ ] 화면 중앙에 가까운 건물 1개만 텍스트로 보이게 하는 기준을 정리한다.
- [ ] 화면 안에 있지만 중앙이 아닌 건물은 점으로 남도록 구조를 정리한다.
- [ ] `focusedBuilding`과 `on-screen candidate`의 역할을 분리한다.
- [ ] 이 단계에서는 마커 배치 품질보다 표시 정책 분리를 우선한다.

### 구현 후 사용자 검수

- [ ] 사용자가 세로모드에서 `중앙 건물은 텍스트`, `주변 건물은 점`으로 보이는지 확인한다.
- [ ] 텍스트가 너무 쉽게 사라지는지 확인받는다.
- [ ] 주변 점이 실제로 화면 안 건물과 대응되는지 확인받는다.
- [ ] 사용자 검수 전까지 3단계로 넘어가지 않는다.

## 3단계. 가로모드 전용 표시 정책 추가

### 구현 범위

- [ ] 가로모드를 단순 회전 상태가 아니라 별도 표시 정책으로 다룬다.
- [ ] 같은 위치에서 세로보다 더 많은 건물을 동시에 보여줄 수 있게 한다.
- [ ] 목표 UX를 `중앙 1개 텍스트 + 화면 안 다른 건물 점`으로 가로모드에 맞게 조정한다.
- [ ] 필요하면 가로모드 후보 수, 중심 판정, 점 유지 기준을 따로 둔다.

### 구현 후 사용자 검수

- [ ] 사용자가 같은 위치에서 세로/가로모드를 비교 테스트한다.
- [ ] 가로모드에서 실제로 더 많은 건물이 동시에 보이는지 확인받는다.
- [ ] 한 건물은 텍스트, 다른 건물은 점으로 동시에 보이는지 확인받는다.
- [ ] 사용자 검수 전까지 4단계로 넘어가지 않는다.

## 4단계. 마커 배치 품질 개선

### 구현 범위

- [ ] `ApplyMarkerPlacement()`의 단순 배치를 개선한다.
- [ ] 건물 표면에 박힌 느낌을 줄이고, 읽기 좋은 위치로 마커를 앞으로 당기는 규칙을 넣는다.
- [ ] 가까운 거리/중간 거리에서 보이는 위치 차이를 보정한다.
- [ ] 이 단계에서는 배치 품질만 다루고, 표시 정책은 최소한만 건드린다.

### 구현 후 사용자 검수

- [ ] 사용자가 가까운 거리와 조금 떨어진 거리에서 마커 위치를 확인한다.
- [ ] 가까울 때만 보이고 멀어지면 사라지는 느낌이 줄었는지 확인받는다.
- [ ] 마커가 실제 건물 앞쪽에 더 자연스럽게 보이는지 확인받는다.
- [ ] 사용자 검수 전까지 5단계로 넘어가지 않는다.

## 5단계. 거리별 표현 전략 정리

### 구현 범위

- [ ] 가까운 거리, 중간 거리, 더 먼 거리에서 어떤 형태로 보여줄지 규칙을 정리한다.
- [ ] 필요하면 점, compact label, selected text를 거리별로 나눈다.
- [ ] 세로/가로모드에서 공통 규칙과 별도 규칙을 구분한다.

### 구현 후 사용자 검수

- [ ] 사용자가 거리 변화에 따라 표시가 자연스럽게 바뀌는지 확인한다.
- [ ] 멀어졌다고 아무 표시도 사라지지 않는지 확인받는다.
- [ ] 최종적으로 원하는 UX에 가까워졌는지 피드백을 받는다.

## 단계별 진행 규칙

- [ ] 한 번에 한 단계만 구현한다.
- [ ] 단계 완료 후 내가 변경 내용을 설명한다.
- [ ] 사용자가 실제 기기 테스트를 진행한다.
- [ ] 사용자 피드백을 반영한 뒤 다음 단계로 넘어간다.
