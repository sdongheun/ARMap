# AR UI / Detail UI Split

이 문서는 현재 프로젝트에서 `AR 환경 UI` 와 `상세정보 UI` 를 역할별로 분리해 설계하기 위한 기준 문서다.

## Summary

- `AR 환경 UI` 는 `uGUI(Canvas) + TMP + 런타임 오버레이` 중심으로 유지한다.
- `상세정보 UI` 는 `AR 환경을 벗어난 별도 페이지` 에서 `UI Toolkit` 으로 구성한다.
- 목적은 `AR 화면은 탐색과 선택에 집중`, `상세 페이지는 읽기와 행동에 집중` 하도록 역할을 나누는 것이다.

## Why Split

- AR 화면은 실시간으로 상태가 바뀌고, 사용자 시선과 카메라 방향에 즉시 반응해야 한다.
- 상세정보 화면은 정적인 정보 레이아웃, 카드 구성, 스크롤, 버튼 설계가 더 중요하다.
- 두 성격을 같은 화면에서 동시에 처리하면 터치 동선과 정보 위계가 쉽게 꼬인다.
- 현재 프로젝트에서 가장 큰 문제였던 `상단 라벨 터치 부담`, `AR 위 상세 오버레이의 복잡함` 을 줄일 수 있다.

## AR Environment UI

### Goal

- 사용자가 주변 건물을 빠르게 탐색하고 선택하게 한다.
- AR 화면은 가능한 한 가볍고 즉시 반응해야 한다.

### Recommended Tech

- `uGUI(Canvas)`
- `TextMeshPro`
- 현재 `ARUIManager`, `GeospatialManager`, `BuildingMarker` 중심 구조 유지

### Elements To Keep In AR

- 상태 안내 문구
  - 예: `앵커가 표시되고 있습니다`, `주변 앵커를 둘러보세요`
- 2D 마커 / 핀
- 중앙 포커스 기준 건물 선택 로직
- 간단 건물 라벨 또는 하단 퀵카드
- `상세 보기` 진입 버튼

### Recommended AR Layout

- 상단: 상태 안내 문구
- 중앙/공간: 마커 및 건물 인지 요소
- 하단: 선택된 건물의 간단 정보
  - 건물명
  - 카테고리
  - 거리 또는 짧은 설명
  - `상세 보기`

### AR UI Principles

- 한 손으로 조작 가능해야 한다.
- 상단 터치보다 하단 터치를 우선한다.
- AR 위에는 긴 텍스트와 많은 카드형 정보를 두지 않는다.
- 즉시 반응, 시인성, 선택 확신이 중요하다.

## Detail Information UI

### Goal

- 사용자가 선택한 건물의 정보를 읽고, 바로 행동하게 한다.
- AR 문맥과 분리된 안정적인 읽기 환경을 제공한다.

### Recommended Tech

- `UI Toolkit`
- `UIDocument + UXML + USS + Controller Script`

### Recommended Transition

- AR 화면 하단 퀵카드의 `상세 보기` 버튼 선택
- AR 화면에서 별도 상세 페이지로 전환
- 뒤로 가기 시 다시 AR 화면 복귀

### Minimum Detail Elements

- 헤더
  - 카테고리
  - 건물명
- 본문
  - 주소
  - 영업시간
  - 전화번호
- 액션
  - 전화
  - 공유
  - 카카오맵 이동

### Detail Page Principles

- 스크롤 없이 주요 정보가 1차로 들어오게 구성한다.
- 정보 카드와 액션 버튼은 앱형 레이아웃으로 정돈한다.
- AR 오버레이처럼 보이게 하지 않고, 독립된 정보 페이지처럼 설계한다.
- 버튼은 `읽기` 보다 `행동` 을 유도하는 역할이 명확해야 한다.

## Final Role Split

### uGUI In AR

- 상태 안내 문구
- 앵커 안내
- 건물 탐색용 2D 마커
- 간단 라벨 또는 하단 퀵카드
- AR 내 선택 흐름 제어

### UI Toolkit In Detail Page

- 건물 상세 헤더
- 주소 / 영업시간 / 전화번호 카드
- 공유 / 전화 / 지도 이동 버튼
- 앱형 상세 레이아웃 전체

## Recommended User Flow

1. 사용자가 AR 화면에서 주변 건물을 탐색한다.
2. 중앙 포커스 또는 선택 기준에 맞는 건물이 감지된다.
3. 하단 퀵카드에 간단 정보가 표시된다.
4. 사용자가 `상세 보기` 를 누른다.
5. AR 화면을 벗어난 상세 페이지가 열린다.
6. 주소 확인, 전화, 공유, 카카오맵 이동 등의 행동을 수행한다.
7. 뒤로 가기로 AR 화면으로 복귀한다.

## Expected Benefits

- AR 화면이 복잡해지지 않는다.
- 상세정보 읽기 가독성이 올라간다.
- 상단 터치 부담이 줄어든다.
- 유지보수 시 `AR UI` 와 `상세 페이지 UI` 를 독립적으로 다듬을 수 있다.
- `uGUI` 와 `UI Toolkit` 의 강점을 각각 맞는 곳에 쓸 수 있다.

## Notes

- AR 안에서의 상세 오버레이는 최소화한다.
- 상세 페이지는 `앱 화면` 처럼 생각하고 설계한다.
- 향후 확장 시에도 `AR 탐색` 과 `정보 소비` 는 분리된 상태를 유지하는 것이 좋다.
